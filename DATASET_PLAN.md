# Dataset Plan (Current) - Unified V3 Metadata

This document reflects the current active plan for full reruns and subset-based robustness analysis.

Primary references:
- [build_dataset_metadata_v3.py](build_dataset_metadata_v3.py)
- [dataset_metadata_v3.csv](dataset_metadata_v3.csv)
- [generate_dataset.py](generate_dataset.py)
- [run_simulation_core.sh](run_simulation_core.sh)
- [build_fd_tensors.py](build_fd_tensors.py)

## Scope

- Total scenarios: 1000
- Scenario IDs: 1..1000 (three-digit filenames)
- Per scenario simulation files: 16 TX inputs (`tx01`..`tx16`)
- Core outputs:
  - S-parameters: `sparams/scenario_XXX.s16p`
  - Frequency-domain tensors: `fd_tensors/scenario_XXX_fd.npz`
  - Normalization stats: `fd_tensors/normalization_freq_full.npz`

## Active Metadata Source

Use [dataset_metadata_v3.csv](dataset_metadata_v3.csv) as the source-of-truth for new full reruns.

It is generated by [build_dataset_metadata_v3.py](build_dataset_metadata_v3.py), which:
- keeps one unified 1000-scenario dataset
- ensures both classes are present in train/val/test
- includes explicit nominal baseline anchors
- includes broad variability coverage for robustness studies

## Split and Class Policy (V3)

Target split ratio per class:
- Train: 70%
- Validation: 15%
- Test: 15%

Current resulting counts in v3:
- Train: y=0 -> 210, y=1 -> 490
- Validation: y=0 -> 45, y=1 -> 105
- Test: y=0 -> 45, y=1 -> 105

Notes:
- Dataset remains intentionally lesion-heavy overall (700 lesion, 300 no-lesion).
- Evaluate with balanced metrics (for example balanced accuracy and F1), not raw accuracy alone.

## Variability Design for Analysis

Controlled factors for robustness and generalisation analysis:
- head scale (`head_scale`)
- head rotation (`head_rotation_deg`)
- measurement noise level (`noise_level`)

V3 includes two profiles:
- `analysis_profile = baseline_nominal`
  - all `N1_baseline` no-lesion samples are nominal reference cases
  - `head_scale = 1.0`, `head_rotation_deg = 0.0`, `noise_level = low`
- `analysis_profile = varied`
  - distributed variability coverage
  - `head_scale` levels: 0.90, 0.95, 1.00, 1.05, 1.10
  - `head_rotation_deg` levels: -15, -10, -5, 0, 5, 10, 15
  - `noise_level`: low, medium, high

## Geometry and Physics Policy

- `antenna_offset` is deprecated in active generation.
- Antenna ring is fixed at nominal placement.
- `head_scale` and `head_rotation_deg` are applied to anatomy and lesion placement.
- Antenna system remains fixed as the measurement reference frame.
- Material perturbations remain metadata-driven through epsilon/sigma variation fields.

## Noise Policy

Noise is injected after S-parameter conversion in [build_fd_tensors.py](build_fd_tensors.py), not during EM simulation.

- Applied to real and imaginary tensor channels.
- Gaussian noise scale uses per-channel standard deviation.
- Noise factors by `noise_level`:
  - low: 0.001
  - medium: 0.005
  - high: 0.01
- Deterministic seeding is used for reproducibility.

## Tensor Format (Current)

Outputs from [build_fd_tensors.py](build_fd_tensors.py):
- `signal`: shape `(512, F)`
- `channels`: full S-matrix real/imag channel names

Details:
- Full 16x16 S-matrix is used (not diagonal-only)
- Real/imag interleaving over 256 complex channels -> 512 real-valued channels
- Train-only normalization stats are fitted and reused for all splits

## End-to-End HPC Flow

1. Prepare metadata:
   - Build or refresh v3 metadata:
     - `python build_dataset_metadata_v3.py`
   - Use `dataset_metadata_v3.csv` as the metadata file for generation/runs.

2. Generate inputs (optional standalone step):
   - `python generate_dataset.py --all --metadata dataset_metadata_v3.csv`

3. Run full sequential simulation pipeline:
   - CPU entrypoint: [run_simulation.sh](run_simulation.sh)
   - GPU entrypoint: [run_simulation_gpu.sh](run_simulation_gpu.sh)

4. Per scenario, [run_simulation_core.sh](run_simulation_core.sh) executes:
   - input generation
   - 16 TX simulations
   - `.out -> .s16p`
   - `.s16p -> fd_tensors/scenario_XXX_fd.npz`
   - cleanup of `.in`/`.out` (by default)

5. After range completion:
   - refresh train-only normalization stats

## Compatibility Notes

- Legacy files (existing `.s16p` and old `_fd.npz`) remain readable.
- New full reruns should use v3 metadata for consistent robustness analysis.
- Keep legacy metadata files for traceability; do not overwrite without backup.

## Recommended Experiment Structure

With one unified dataset, train/evaluate subsets by filtering `analysis_profile` and factor levels:
- nominal baseline only
- rotation sweeps
- scale sweeps
- noise sweeps
- combined stress tests

This supports reproducible performance-vs-variation reporting from a single master dataset.
